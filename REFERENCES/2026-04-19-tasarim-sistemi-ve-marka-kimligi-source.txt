Sen uzman bir UI/UX Tasarımcısı ve Marka Stratejistisin. Hedef kitlesi teknolojiye mesafeli olan kullanıcılarda "güven, netlik ve erişilebilirlik" duygularını uyandıran modern web arayüzleri kurgulama konusunda uzmansın.

Thought for 22m 40s
EsnafDigital — Design System + Brand DNA
Aşağıdaki sistemin ana fikri şu: az gürültü, net aksiyon, yüksek okunabilirlik ve güven veren sade modernlik.

2) Tipografi
Google Fonts tarafında Inter, ekranlar için tasarlanmış bir variable font; yüksek x-height’i küçük ve orta boy metinlerde okunabilirliği destekliyor, ayrıca tabular numbers desteği panel sayıları ve fiyat gösterimlerinde avantaj sağlıyor. Next.js dokümantasyonu da next/font ile Google fontlarını optimize etmeyi ve birden fazla fontu gereksiz yere çoğaltmamayı öneriyor. Bu yüzden çekirdek sistemde tek aile kullanıyorum: Inter.

3) UI Bileşen Kuralları
3.1 Butonlar
W3C, tıklanabilir hedeflerin en az 44×44 CSS px olmasını ve klavye odağının görünür kalmasını öneriyor. Bu yüzden bu sistemde ana buton ve input yüksekliğini 48px, icon-only aksiyonları ise en az 44px tutuyorum; ayrıca :focus-visible ile belirgin bir ring zorunlu. 

Şekil kararı:
Tam yuvarlak değil, hafif yuvarlatılmış.
Yani:

Buton / input: 12px

Kart / modal: 16px

Badge / chip: 9999px

Neden?
Sivri köşeler sert ve mesafeli görünür. Tam pill butonlar ise fazla “app-like” ve oyuncu hissettirir. 12px radius, EsnafDigital için en iyi orta noktadır: güvenli, modern, ama ciddi.

Buton varyant standardı
Varyant	Default	Hover	Active	Disabled
Primary	brand-600 zemin, beyaz metin	brand-700	brand-700, shadow azalır	line-strong zemin, ink-disabled
Secondary	beyaz zemin, line-strong border, ink-title	surface-subtle	border koyulaşır, shadow yok	border açık, metin disabled
Ghost	transparan, brand-700 metin	brand-50 zemin	brand-100	düşük opaklık
Destructive	error fill, beyaz metin	daha koyu kırmızı	shadow kalkar	soluk kırmızı-gri
Buton davranışı
Hover: sadece bir ton koyulaşma + hafif shadow artışı

Active: zıplama yok; sadece shadow azalır ve basılmış hissi verir

Transition: 150–180ms

Focus: 4px ring (brand-100 tabanlı)

Icon + text kombinasyonu varsayılan olsun; sadece evrensel ikonlarda icon-only kullan

CTA metin kuralı:
Buton etiketi genel değil, sonucu söyleyen fiil içersin.

Kaçınılacak: Devam Et

3.2 Kartlar ve Formlar
Kart standardı
Background: surface

Border: 1px solid line

Radius: 16px

Shadow: yumuşak ve sığ

Padding: 20px mobil / 24px desktop

Shadow standardı

shadow-soft: 0 1px 2px rgba(15,23,42,0.04), 0 8px 24px rgba(15,23,42,0.06);
shadow-soft-hover: 0 2px 6px rgba(15,23,42,0.08), 0 14px 32px rgba(15,23,42,0.10);
Kural:
Bu üründe border-first, shadow-second yaklaşımı kullanılmalı.
Yani kartın ayrımı önce net bir kenar çizgisiyle, sonra hafif gölgeyle yapılmalı. Ağır gölgeler güven yerine “reklam kutusu” hissi verebilir.

Dashboard kartları
Daha fonksiyonel, daha sıkı

Başlık + kısa açıklama + ana veri + tek aksiyon

Bir kartta en fazla 1 ana metrik + 1 ikincil bilgi + 1 aksiyon

Vitrin site kartları
Aynı radius ve border mantığı

Daha ferah boşluk

İkon + başlık + tek paragraf + CTA düzeni

5) Doğrudan Kullanılabilir Next.js / Tailwind Yapısı
Next.js tarafında fontları next/font ile bağlamak en temiz yaklaşım; yerleşik next/font fontları optimize eder, self-host eder, dış istekleri kaldırır ve CSS variable kullanımını destekler. Subset tanımı da preload davranışı için önemlidir. Tailwind tarafında da renk, tipografi, radius ve shadow token’ları theme katmanında tanımlanabilir; aynı mantık Tailwind 4’te @theme değişkenleriyle taşınır.

“Next.js + Tailwind + shadcn uyumlu component blueprint” formatında çıkarayım; her component için props, states, className ve örnek JSX ile.


Harika! Çıkardığın Component Inventory, Layout Sistemi ve UX kararları EsnafDigital'in ruhunu tam olarak yakalamış. Sade, güven veren ve doğrudan hedefe odaklanan bu yapıyı çok beğendim.

Lütfen teklif ettiğin gibi bir sonraki adıma geçelim. Bana 'İlk Sprint İçin Öncelikli Component Seti'nde belirttiğin Marketing MVP ve Dashboard MVP bileşenlerini, Next.js + Tailwind CSS + shadcn/ui standartlarına uygun, doğrudan projeye entegre edebileceğim Component Blueprint formatında çıkar.

Her bileşen için şunları mutlaka dahil et:

Types/Interfaces: Bileşenin alacağı props tanımları.

  onSubmit?: (values: ContactFormValues) => Promise<void> | void
  className?: string

const defaultBusinessTypes: SelectOption[] = [
  { label: "Berber", value: "barber" },
  { label: "Güzellik Salonu", value: "beauty" },
  { label: "Kafe / Restoran", value: "cafe" },

]

export function ContactForm({
  title = "Size ulaşalım",
  description = "Numaranızı bırakırsanız size uygun bir zamanda geri dönüş yapabiliriz.",
  submitLabel = "Size Ulaşalım",
  businessTypes = defaultBusinessTypes,
  defaultValues,
  onSubmit,
  variant,
  className,
}: ContactFormProps) {
  const [isPending, setIsPending] = React.useState(false)
  const [form, setForm] = React.useState<ContactFormValues>({
    fullName: defaultValues?.fullName ?? "",
    businessName: defaultValues?.businessName ?? "",
    phone: defaultValues?.phone ?? "",
    businessType: defaultValues?.businessType ?? "",
    note: defaultValues?.note ?? "",
  })

  function updateField<K extends keyof ContactFormValues>(
    key: K,
    value: ContactFormValues[K]
  ) {
    setForm((prev) => ({ ...prev, [key]: value }))
  }

  async function handleSubmit(event: React.FormEvent<HTMLFormElement>) {
    event.preventDefault()
    setIsPending(true)

    try {
      if (onSubmit) {
        await onSubmit(form)
      }
    } finally {
      setIsPending(false)
    }
  }

  return (
    <form
      onSubmit={handleSubmit}
      className={cn(contactFormVariants({ variant }), "space-y-5", className)}
    >
      <div className="space-y-2">
        <h3 className="text-h3 font-semibold text-ink-title">{title}</h3>
        <p className="text-body text-ink-secondary">{description}</p>
      </div>

      <div className="grid gap-4 md:grid-cols-2">
        <Input
          id="fullName"
          label="Ad soyad"
          value={form.fullName}
          onChange={(e) => updateField("fullName", e.target.value)}
          placeholder="Adınız soyadınız"
          required
        />

        <Input
          id="businessName"
          label="İşletme adı"
          value={form.businessName}
          onChange={(e) => updateField("businessName", e.target.value)}
          placeholder="İşletmenizin adı"
          required
        />
      </div>

      <div className="grid gap-4 md:grid-cols-2">
        <Input
          id="phone"
          type="tel"
          label="Telefon numarası"
          value={form.phone}
          onChange={(e) => updateField("phone", e.target.value)}
          placeholder="05xx xxx xx xx"
          required
        />

        <Select
          id="businessType"
          label="İşletme türü"
          value={form.businessType}
          onChange={(e) => updateField("businessType", e.target.value)}
          defaultValue=""
          placeholder="Bir seçim yapın"
          options={businessTypes}
          required
        />
      </div>

      <Textarea
        id="note"

        placeholder="İşletmeniz veya ihtiyacınızla ilgili kısa bir not bırakabilirsiniz."
      />

      <Button type="submit" isLoading={isPending} fullWidth>
        {submitLabel}
      </Button>
    </form>
  )

function Container({
  children,
  className,
}: {
  children: ReactNode
  className?: string
}) {
  return (
    <div className={cn("mx-auto w-full max-w-6xl px-4 md:px-6", className)}>
      {children}
    </div>
  )
}

function SectionHeading({
  eyebrow,
  title,
  description,
  className,
}: {
  eyebrow?: string
  title: string
  description?: string
  className?: string
}) {
  return (
    <div className={cn("space-y-3", className)}>
      {eyebrow ? (
        <p className="text-caption font-semibold uppercase tracking-[0.12em] text-brand-700">
          {eyebrow}
        </p>
      ) : null}

      <h2 className="max-w-[18ch] text-h2 font-bold tracking-[-0.02em] text-ink-title">
        {title}
      </h2>

      {description ? (
        <p className="max-w-[68ch] text-body text-ink-secondary">{description}</p>
      ) : null}
    </div>
  )
}
