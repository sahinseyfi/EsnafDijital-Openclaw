## Gece baslangic notu
- Business detail icindeki scrape hazirlik paneli icin iki mod karari netlesti: varsayilan `Hafif tarama`, operator acarsa `Derin tarama`.
- `Hafif tarama` tek isletme icin dusuk maliyetli audit on hazirligi olacak: website kontrolu, Google Search gorunurluk aramasi, Google sonuc sayfalarindan temel bilgi toplama ve mevcut Google Maps/discovery snapshot okuma.
- `Derin tarama` ise secmeli ve maliyet uyarili genis kaynak katmani olacak: Google Maps detay yenileme, Instagram, Yandex, Apple Maps ve ek zenginlestirme.

## Plugin ve scrape arastirmasi
- Scrape tarafini guclendirebilecek bundled adaylar `firecrawl`, `exa`, `browser`, `duckduckgo`, `brave` olarak kontrol edildi; bu ortamda allowlist disinda kaldiklari icin aktif kullanima gecmedi.
- Kisa maliyet sonucu: `Firecrawl` tek seferlik sinirli free kredi ile basliyor, `Exa` esasen ucretli, `Brave` API key gerektiriyor; bu yuzden yakin vadede ucretsiz/guvenilir omurga `internal website check + mevcut web arastirmasi + Apify` cizgisinde kaldi.

## Agent browser kurulumu
- Supheli gorunen browser skill adaylari elendi; temiz aday olarak `agent-browser-core` secildi ve workspace'e kuruldu. Commit: `52d1941` (`Install agent-browser-core skill`).
- Alt runtime da kuruldu: `agent-browser@0.26.0` global npm ile yuklendi, `agent-browser install` ile Chrome binary indirildi, Linux bagimlilik eksigi nedeniyle `agent-browser install --with-deps` calistirildi.
- Smoke test gecti: `agent-browser open https://example.com --session smoke-test` ve `agent-browser get title --session smoke-test` calisti, sonra session kapatildi.
- Host tarafinda ek not: kurulum sonunda kernel upgrade uyarisi geldi (`6.8.0-110-generic` bekleniyor); browser runtime icin engel degil ama host reboot planlanabilir.

## Business detail referans birlestirme

- Yeni dokuman business detail sayfasinin rolunu, veri katmanlarini, UX akisini, scrape paneli cizgisini ve V1/V2 acilis sirasini tek yerde topluyor.

## Refresh route kapanis notu
- `refresh-discovery` 500'unun asil nedeni Apify komutunun kendisi degil, route icindeki `execFile` bekleme hattiydi; saf shell ayni komutu `41s` civarinda tamamlarken `execFile` child pipe'larini acik tuttugu icin timeout'a gidiyordu.
- Route `spawn + stdio ignore` cizgisine alindi ve timeout `300s` olarak korundu; build tekrar gecti.
- Izole runtime smoke testinde `Berber Serkan` kaydina POST `/api/businesses/:id/refresh-discovery` `200` dondu, yeni snapshot yazildi ve Business Detail sayfasinda `23 Nis 03:00` snapshot zamani ile timeline girdisi gorundu.
- Ardindan ana runtime deploy edildi, `3012` uzerinde ayni endpoint tekrar `200` verdi ve blocker kapandi.
